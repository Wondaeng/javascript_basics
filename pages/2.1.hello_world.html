Hello, World!

alert을 이용하여 JS 버전의 Hello, world!를 만들어보겠습니다. 사실 이미 만들어져있습니다. 이 페이지를 클릭하는 순간 알림이 뜨지 않았었나요? 참고로. alert은 브라우저 환경에서만 사용가능한 커맨드입니다. Node.js와 같은 다른 자바스크립트 환경에서는 동작하지 않습니다.
그럼 먼저, 어떻게 스크립트를 웹페이지에 써넣을 수 있는지에 대해 먼저 알아보도록 하겠습니다. (참고: Node.js 같은 서버 환경에서는 "node my.js"와 같이 명령어를 입력하여 스크립트를 실행할 수 있습니다.)

"script" 태그

JS 프로그램은 HTML 문서에 <script> 태그를 이용해 넣을 수 있습니다. 예를 들어, 

        
            <!DOCTYPE HTML>
            <html>
            <body>
                <p>Before the script...</p>

                <script>
                    alert('Hello, world!');
                </script>

                <p>...After the script.</p>
            </body>
            </html>
이 <script> 태그는 브라우저가 <script> 태그를 처리함과 동시에 자동으로 실행됩니다. 이 페이지의 소스코드(우클릭 후 페이지 소스보기)를 보시면 해당 스크립트가 10-12번째 줄에 삽입 되어있음을 알 수 있습니다. 때문에, 페이지를 로드하자마자 "Hello, world!"라 쓰인 알림창을 마주하게 된 것입니다.

현대 마크업

과거에는 <script>에 속성(attribute)으로 <script type="text/javascript">와 같이 type을 명시해줘야 했었습니다. 하지만 오늘날 현대의 HTML 표준에선 더이상 필요하지 않으며, type 속성의 의미도 완전히 바뀌었습니다 (이 내용은 심화 과정으로 나중에 다룹니다).
또한, 이 스크립트가 어떤 언어인지를 나타내는 <script language=...> 속성도 있었습니다. 물론 이젠 JS가 기본 언어이므로 더이상 사용되지 않습니다.

External scripts

위에서 설명한 것 처럼 HTML에 <script> 태그를 쓰고, 그 안에 직접 스크립트를 작성하여 JS 프로그램을 실행시킬 수도 있지만, src 속성을 통해 해당 경로의 .js 파일을 불러와 삽입할 수도 있습니다: 
            
                <script src="/path/to/script.js"></script>
여기서 /path/to/script.js는 "절대(absolute)"경로입니다. src="script.js"나 src="./script.js"와 같이 (현재 브라우저가 읽고있는 HTML 파일의 경로를 working directory로 했을 때의) 상대(relative) 경로를 사용해도 됩니다. 
            
                <script src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.11/lodash.js"></script>
처럼 URL로 JS 스크립트를 불러올 수도 있습니다.
그럼 언제 HTML에 직접 쓰고 언제 별도의 파일로 불러올까요? 일반적으로 스크립트가 아주 간단한 경우에만 직접 쓰고, 그 외의 경우엔 별도의 파일로 작성한뒤 불러오는 것이 표준입니다. 특히, 별도의 파일로 불러오는 경우엔 해당 .js 파일이 브라우저에 의해 캐싱되어서, 다른 페이지에서 똑같은 파일을 불러오는 경우 추가적으로 다시 다운로드 하지 않고 바로 해당 스크립트를 불러올 수 있다는 장점이 있습니다. 이는 사이트의 트래픽을 줄이고, 로딩을 더 빠르게 합니다.
또, <script> 태그는 src 속성을 사용하여 스크립트를 불러오는 것과, <script> 태그 사이에 스크립트를 직접 입력하는 것을 동시에 할 수 없습니다. 두가지가 모두 써있는 경우에는 src를 우선으로 하고, 직접 작성한 스크립트는 무시됩니다.
